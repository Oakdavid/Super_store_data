# Super Store Normalized

This focuses on importing, transforming, and modeling the **Super Store Normalized dataset** in **Power BI**. 
The aim is to create an **optimized data model** for business reporting and analysis, removing unnecessary fields and combining related tables to improve efficiency.

## Steps Taken
1. **Data Examination:** Analyzed all tables to determine relationships between fact and dimension tables.
2. **Dimension Table Optimization:** Combined related dimension tables into single tables and disabled unused tables from loading.
3. **Column Cleanup:** Removed unnecessary columns, including `Order ID`, to streamline the model.
4. **Data Transformation:** Imported all worksheets into Power Query, ensuring standard formatting, proper data types, and no duplicates.
5. **Model Relationships:** Confirmed that relationships between fact and dimension tables are correctly established.
6. **Testing:** Verified model integrity by creating visuals using fields from both fact and dimension tables.
   
## Project Features
- Optimized data model for performance and clarity
- Fact and dimension tables properly structured
- Cleaned and merged datasets for easy reporting
- Ready-to-use for interactive Power BI dashboards
